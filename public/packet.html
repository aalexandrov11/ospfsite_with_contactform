OSPF пакет

Всички пакети OSPF споделят обща структора от 24-байта. Тя позволява на получаващият рутер да валидира и обработва пакетите:

Структора на OSPF пакет
Version # Type Packet lenght Router ID Area ID Checksum Authentication type Authentication Data
  • Version # - Версията обикновено е №2. (1 байт)
  • Type - Той определя типа OSPF пакет. Има 5 различни типа пакети OSPF. (1 байт)
    •    
    1. Hello пакет
      2.
    2. Database Descriptor пакет
      3.
    3. Link State Request пакет
      4.
    4. Link State Update пакет
      5.
    5. Link State Acknowledgment пакет
      6.

  • Packet lenght - Обща дължина на OSPF пакета (2 байта)
  • Router ID - Идент. № на рутера който изпраща пакета (4 байта)
  • Area ID - 32-битов идентификатор на зона, присвоен на интерфейса, изпращащ OSPF пакета (4-байта)
  • Checksum - Стандартна IP контролна сума на OSPF пакет, с изключение на полето за удостоверяване (2-байта)
  • Authentication type - Тип на удостоверяване (2-байта)
    •    
    1. Без парола
      2.
    2. Текстова парола
      3.
    3. MD5 (хеширащ алгоритъм) автентификация
      4.

  • Authentication - Автентификационни данни за потвърждаване целостта на пакета (8-байта)
  • Data - Може да съдържа всякаква информация и няма фиксирана дължина

Типове пакети

Tип пакет Функция
Hello пакет Изпраща се периодично за откриване и поддържане на отношенията на съседите.
Database Descriptor (DD) пакет Съдържа кратка информация за локалната база данни за състояние на връзки (LSDB) и синхронизира LSDB на две устройства.
Link State Request (LSR) пакет Изисква необходимите LSA (link-state advertisement) от съседите. LSR пакетите се изпращат само след успешна обмяна на DD пакети.
Link State Update (LSU) пакет Изпраща необходимите LSA на съседите.
Link State Acknowledgment (LSAck) пакет Признава получаването на LSA.
